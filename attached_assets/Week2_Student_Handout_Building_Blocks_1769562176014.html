The Three Layers of Every App

Every app-whether built with Replit, Streamlit, or any other tool-has three fundamental layers:

Layer

What It Does

Restaurant Analogy

STRUCTURE

(HTML / Layout)

The content and organization. Headings, paragraphs, buttons, images, forms. The menu items, tables, chairs-the physical "stuff" in the restaurant.

STYLE

(CSS / Design)

The visual appearance. Colors, fonts, spacing, layout, animations. The decor, lighting, table cloths-how the restaurant looks and feels.

BEHAVIOR

(JS / Python)

The interactivity. Calculations, form submissions, dynamic updates, API calls. The waiters, kitchen, ordering system-what makes things happen.

Platform Differences

In Replit/Lovable web apps, these are HTML, CSS, and JavaScript. In Streamlit, Python handles all three but you'll still see structure (st.title, st.columns), style (st.markdown with CSS), and behavior (calculations, button clicks).

2. The Styling (Tailwind CSS Classes)

className="md:hidden p-2 text-foreground"

Class

What It Does

md:hidden

Hides this button on medium screens and larger (tablets/desktops). It only shows on mobile!

p-2

Adds padding (spacing inside the button)

text-foreground

Sets the icon color to your theme's foreground color

In plain English: "This button only appears on phones. On bigger screens, users see the full navigation menu instead."

3. The Click Action

onClick={() => setIsOpen(!isOpen)}

Part

What It Means

onClick

"When someone clicks this button..."

setIsOpen()

"...update the isOpen variable..."

!isOpen

"...to the opposite of what it currently is"

In plain English:

  • the menu is closed (isOpen = false), clicking makes it open (isOpen = true)
  • the menu is open (isOpen = true), clicking makes it closed (isOpen = false)

It's a toggle switch!

Activity 2: The 4-Step Debugging Process

Goal: Learn to describe problems so AI can fix them

When something breaks, DON'T just say "it's not working." Use this process:

1

WHAT'S WRONG

Describe what's actually happening. "The button doesn't do anything when I click it."

2

WHAT'S EXPECTED Describe what SHOULD happen. "It should calculate the total and display it below."

3

SHARE CODE Include the relevant code. If unsure which part, share it all.

4

ASK CLEARLY "Please fix this bug" or "Why isn't this working?"

MASTER DEBUGGING PROMPT TEMPLATE

I have a bug in my app.

WHAT'S HAPPENING: When I click the Calculate button, nothing

appears on screen. No error message, just nothing.

WHAT I EXPECT: The total price should appear below the button

showing quantity price.

HERE'S MY CODE:

[paste your code here]

Please help me fix this bug.

Activity 3: Make It Mobile-Friendly

Goal: Ensure your app works on phones

1

Test on Mobile

  • Resize your browser window to phone-width (~375px)
  • Or use browser dev tools: Right-click Inspect Toggle device toolbar
  • Check: Can you read everything? Do buttons work? Does layout break?

2

Identify Problems

  • Text too small to read?
  • Buttons too small to tap?
  • Content cut off or overlapping?
  • Horizontal scrolling required?

3

Ask AI to Fix

  • Describe each mobile issue you found
  • Ask for responsive design improvements
  • Test again after each fix

MOBILE FIX PROMPT

My app doesn't look good on mobile. Please make these fixes:

  1. The navigation menu overlaps the content on small screens
  2. The text is too small to read on phones
  3. The buttons are too close together to tap accurately
  4. The two-column layout should stack vertically on mobile

Make the design responsive for screens from 320px to 1920px wide.

Key Takeaways

1

Read, Don't Write

You don't need to write code from scratch-but reading it helps you communicate problems and request changes more effectively.

2

Describe Problems Completely

Good bug reports include: what's happening, what should happen, and the relevant code. This gets you fixes faster.

3

Mobile-First Mindset

More than half of web traffic is mobile. If your app doesn't work on phones, you've lost half your audience.

4

Forms Create Value

Static content informs. Forms and inputs transform information. That's where business tools start.
